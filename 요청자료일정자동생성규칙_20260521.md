# 요청자료일정 자동 생성 규칙

작성일: 2026-05-21
작성: COSMAX BTI 경영관리팀
대상 시스템: 월간일정표 대시보드 (App Runner / GitHub Pages)
적용 시점: 2026년 6월 이후 일정부터

---

## 1. 개요

월간 요청자료 일정 중 정형화된 항목을 그룹 일정표의 확대회의 등록 시 자동으로 생성·갱신·삭제하기 위한 규칙을 정의한다. 본 규칙은 `app.js`의 `syncAutoRequestForMonth()` 함수와 `_backfill_auto_request.py` 백필 스크립트에 동일하게 구현되어 있다.

## 2. 자동 생성 항목 일람

| 순번 | 항목명 | 일자 산정 기준 | 카테고리 (칩 색상) |
|----|---|---|---|
| 1 | 공통전략지표 전월 마감 업데이트 | 해당 월 4번째 영업일 | 정기요청자료 (청록) |
| 2 | 팀즈[공통지표관리채널] Upload | 해당 월 4번째 영업일 | 정기요청자료 (청록) |
| 3 | 경영실적(예상) 엑셀자료 회신 | 확대회의 D-4 영업일 | 통합회의및확대회의관련 (파랑) |
| 4 | 확대회의 PPT자료 회신 (NBT/BIO/펫/파마) | 확대회의 D-3 영업일 | 통합회의및확대회의관련 (파랑) |
| 5 | 운전자본(채권/재고/재무제표) 자료 회신 | 해당 월 9번째 영업일 | 정기요청자료 (청록) |
| 6 | 확대회의 회의록 회신 | 확대회의 D+1 영업일 | 통합회의및확대회의관련 (파랑) |
| 7 | 가마감예상실적(매출) 자료 회신 | 해당 월 영업말일 D-4 영업일 | 정기요청자료 (청록) |

D = 그룹 확대회의 기준일 (해당 월 그룹 일정에서 가장 빠른 확대회의 일자)

## 3. 영업일 정의

- 영업일 = 평일 ∧ ¬법정공휴일 ∧ ¬대체공휴일
- 토요일·일요일 제외
- 한국 법정공휴일 + 임시공휴일(지방선거 등) + 대체공휴일 모두 회피
- 휴일 데이터 출처: `app.js`의 `holidays` 사전 (양 시스템 동일)

### 3.1 N번째 영업일 산정

해당 월 1일부터 평일을 순회하며 휴일을 제외한 N번째 평일을 반환한다.

예시 (2026년 6월):
- 6/1 (월) = 1
- 6/2 (화) = 2
- 6/3 (수) = 지방선거 → 제외
- 6/4 (목) = 3
- 6/5 (금) = 4 → 공통전략지표·팀즈Upload 일자
- 6/8 (월) = 5
- 6/9 (화) = 6
- 6/10 (수) = 7
- 6/11 (목) = 8
- 6/12 (금) = 9 → 운전자본 일자

### 3.2 영업일 D ± n 계산

기준일 D 부터 ±1 씩 이동하며 휴일·주말을 건너뛰어 n번째 영업일을 반환한다.

예시 (D=2026-06-16 화):
- D-1 = 6/15 (월)
- D-2 = 6/12 (금)
- D-3 = 6/11 (목) → 확대회의 PPT자료 회신
- D-4 = 6/10 (수) → 경영실적(예상)
- D+1 = 6/17 (수) → 확대회의 회의록 회신

### 3.3 영업말일 D-4

해당 월 마지막 날부터 거꾸로 평일·휴일을 제외한 첫 영업일을 영업말일로 정한 뒤, 거기서 -4 영업일을 적용한다.

예시 (2026년 6월):
- 6/30 (화) = 영업말일
- 영업말일 D-4 = 6/24 (수) → 가마감예상실적

## 4. 트리거 조건

| 시점 | 동작 |
|---|---|
| 그룹 일정에 4종 확대회의(NBT/BIO/펫/파마) 중 **1종이라도 추가** | 위 7개 항목 자동 생성 |
| 확대회의 **일자 변경** | 항목 7개의 산정일도 함께 이동 (동일 ID 유지) |
| 해당 월 4종 확대회의가 **모두 삭제** | 7개 항목 자동 삭제 |
| 가마감예상실적 | 회의 유무와 무관, 월 1회 독립 생성 (단 트리거 발화 시 함께 생성) |

## 5. 항목 식별자 (결정적 ID)

월별로 1세트만 존재하도록 결정적 ID를 사용한다. 회의일자가 변경되어도 같은 ID로 upsert되어 충돌 없이 갱신된다.

| 항목 | ID 패턴 |
|---|---|
| 공통전략지표 | `req-auto-{YYYY-MM}-strategy` |
| 팀즈 Upload | `req-auto-{YYYY-MM}-teams` |
| 경영실적(예상) | `req-auto-{YYYY-MM}-revenue-est` |
| 확대회의 PPT자료 회신 | `req-auto-{YYYY-MM}-conf-submit` |
| 운전자본 | `req-auto-{YYYY-MM}-working-capital` |
| 확대회의 회의록 회신 | `req-auto-{YYYY-MM}-conf-minutes` |
| 가마감예상실적 | `req-auto-{YYYY-MM}-month-close` |

## 6. 그룹↔NBT/BIO 미러링 (연관 규칙)

자동 생성과 별개로, 그룹 일정의 확대회의 항목은 다음 규칙에 따라 회사별 일정 및 요청자료에 미러된다.

| 그룹 제목 패턴 | 회사 일정 미러 (schedules) | 요청자료 미러 (request_schedules) |
|---|---|---|
| 엔비티 / NBT 확대 | NBT 회사 | `req-conf-{date}-nbt` |
| 바이오 확대 | BIO 회사 | `req-conf-{date}-bio` |
| 펫 확대 | (없음) | `req-conf-{date}-pet` |
| 파마 확대 | (없음) | `req-conf-{date}-pharma` |

NBT/BIO 미러는 `{Company}-mirror-{date}-{key}` 형식의 결정적 ID를 사용한다.

## 7. 칩 표시 규칙 (요청자료 패널)

원본 데이터(`title`)는 그대로 보존하되, 캘린더 칩의 표시 텍스트만 다음과 같이 축약한다. Hover tooltip 및 모달에는 원본 명칭이 그대로 노출된다.

| 원본 텍스트 | 칩 표시 |
|---|---|
| `(판교)` 우측 위치 표기 | 제거 |
| `(NBT/BIO/펫/파마)` 회사 나열 | 제거 |
| `코스맥스펫` | `펫` |
| `코스맥스파마` | `파마` |

예시:
- `1400 코스맥스펫 확대회의(판교)` → `1400 펫 확대회의`
- `확대회의 PPT자료 회신 (NBT/BIO/펫/파마)` → `확대회의 PPT자료 회신`

## 8. 인쇄(PDF) 출력 규칙

- 인쇄 헤더: 좌측 세로 바 + `{YYYY년 N월}` + `건기식·제약부문 요청자료 일정` / 우측 `COSMAX BTI 경영관리팀`
- 셀 높이는 자동 확장되어 내용 잘림 없음
- 칩 폰트 6.5pt, line-height 1.2 로 축소되어 다수 항목 수용
- 카테고리 색상 강제 출력(`print-color-adjust: exact`) 적용
